1.3 Global CSS Utilities/Components (src/index.css)
- body default: bg-gray-50 text-gray-900 antialiased.
- dark body: bg-slate-950 text-slate-100.

Reusable classes:
- .card: white background, rounded, subtle border/shadow, dark variant supported.
- .btn: base button spacing/inline-flex/transition.
- .btn-primary: primary-600 -> primary-700/800 on hover/active.
- .btn-secondary: gray tones + dark mode variants.
- .btn-success: success-600/700.
- .btn-danger: danger-600/700.
- .icon-button: square icon action button with border/hover/focus ring.
- .input: full-width rounded inputs with focus ring primary-500.
- .badge + badge variants: success/warning/danger/info.

Custom animations:
- emergency-modal-vibrate (horizontal micro vibration loop).
- emergency-case-added (red pulse/highlight animation).
- reduce-motion media query disables those animations.

Dark mode class remapping:
- Overrides common bg/text/border utility classes to slate-scale equivalents.

1.4 Toaster Theme (main.tsx)
- Default toast background #363636, text #fff.
- Success icon theme primary #22c55e.
- Error icon theme primary #ef4444.

1.5 App shell metadata (index.html)
- Theme color meta: #0066cc.
- App favicon + og image: VitaSense logo.

9) IMPLEMENTATION NOTES FOR REACT NATIVE MIGRATION
====================================================================

Web-specific patterns to replace:
- Tailwind classes -> RN StyleSheet/theme tokens.
- HTML tables -> FlatList/SectionList card/table equivalents.
- CSS animations -> Animated API/Reanimated.
- Recharts -> RN chart library equivalent.
- File input/download/PDF DOM APIs -> RN document picker + file system + PDF lib.
- Socket logic can be retained conceptually with socket.io-client RN support.

Most styling relies on:
- Gray background surfaces + white cards + semantic accent colors.
- Badge semantics for status/severity.
- Icon-based affordances and high-density dashboard cards.
